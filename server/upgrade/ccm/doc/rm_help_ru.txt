

Обновление установленной версии RM

Примечания:
      - Этот сценарий обновляет RM. См. раздел справки "Обновление Rational solution for CLM".
      - За исключением вывода данной справки, этот сценарий должен выполняться из каталога JazzInstallRoot/server.

Формат:
    rm_upgrade
         -oldApplicationHome <значение> --Полный путь к каталогу конфигурации сервера RM в предыдущей версии. Путь для обычной установки: <каталог-установки-RM>/server/conf.
[-oldJTSHome <значение>] --Полный путь к каталогу конфигурации сервера JTS в предыдущей версии. Для стандартной установки это <каталог-установки-JTS>/server/conf.
         [-applicationContextRoot rdm] --Корневой контекст RM.
         [-db.jdbc.location <нет>] --Путь к каталогу базы данных RM версии 5.0. Эта база данных должна быть точной копией базы данных JTS из версии 4.0.X.
         [-db.jdbc.password <нет>] --Пароль базы данных JDBC.
         [-db.db2i.schema.name <нет>] --Имя схемы DB2i. Используется только для типа базы данных DB2i.
         [-db.db2.dsn.dbname <нет>] --Имя базы данных DSN DB2. Используется только для типа базы данных DB2z.
         [-db.schemaPrefix <нет>] --Префикс схемы базы данных. Используется только для типа базы данных DB2z.
  		 [-oldApplicationVersion <нет>] --Обновляемая версия. По умолчанию она определяется командой repotools -version, выполняемой для приложения RM предыдущей версии.
         [-updateTomcatFiles <yes (по умолчанию) | no>] --Требуется ли обновлять файлы конфигурации tomcat.
         [-oldApplicationTomcatHome <нет>] --Полный путь к корневому каталогу Tomcat RM в предыдущей версии. По умолчанию значение параметра oldApplicationTomcatHome создается на основе значения параметра oldApplicationHome.
         [-oldJTSTomcatHome <нет>] --Полный путь к корневому каталогу Tomcat JTS в предыдущей версии. По умолчанию значение параметра oldJTSTomcatHome создается на основе значения параметра oldJTSHome.
         [-newApplicationHome conf] --Путь к каталогу конфигурации сервера RM в новой версии.
         [-newJTSHome <нет>] --Полный путь к каталогу конфигурации сервера JTS в новой версии. По умолчанию значение параметра newJTSHome создается на основе значения параметра newApplicationHome.
         [-newJTSContextRoot jts] --Корневой контекст JTS сервера в новой версии.
	     [-newApplicationTomcatHome <нет>] --Путь к домашнему каталогу Tomcat RM в новой версии. По умолчанию значение параметра newApplicationTomcatHome создается на основе значения параметра newApplicationHome.
         [-newJTSTomcatHome <нет>] --Путь к домашнему каталогу Tomcat JTS в новой версии. По умолчанию значение параметра JTSTomcatHome создается на основе значения параметра newJTSHome.
         [-ignoreJTSVersionCheck] --Не проверять, совместима ли версия приложения с версией JTS, расположенной в newJTSHome.  Этот параметр следует использовать только в распределенной среде, где JTS расположен в отдельной системе, которая недоступна из системы приложения.
         [-noPrompt] --Не показывать приглашение перед созданием или изменением таблиц.
         [-noEditor] --Не открывать редактор для файлов teamserver.properties.
         [-noVerify] --Не выполнять проверку файлов teamserver.properties.
         [-noStepPrompt] --Не показывать приглашение перед выполнением операции.

Примеры:
   upgrade/rm/rm_upgrade.sh -oldApplicationHome /opt/JazzTeamServer4_0/rm/server/conf
